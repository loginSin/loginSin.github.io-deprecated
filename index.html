python-urllib2实现网页下载器

使用urllib2来实现网页下载器有3种方法
1：最简单的方法：

#-- coding: UTF-8 --
import urllib2

#直接请求网页
response = urllib2.urlopen(“http://www.baidu.com“)

#获取状态码，如果是200表示成功
print (response.getcode())

#读取内容
cont = response.read()
print (cont);

2:添加data、http header

#-- coding: UTF-8 --
import urllib2

#创建Request对象
url = “http://www.baidu.com“
req = urllib2.Request(url);

#添加数据
req.add_data(‘a’,’1’)

#添加http的header
req.add_header(‘User-Agent’,’Mozilla/5.0’)

#发送请求获取数据
response = urllib2.urlopen(req)
3：添加特殊情景的处理器
HTTPCooKieProcessor(需要cookie访问的网页)
ProxyHandler(需要代理才能访问的网页)
HTPPSHandler(使用https加密访问的网页)
HTTPRedirectHandler(url相互自动跳转的网页)

#-- coding: UTF-8 --
import urllib2,cookielib

#创建cookie容器
cj = cookielib.CookieJar()

#创建一个opener
opener = urllib2.build_opener(urllib2.HTTPCookieProcessor(cj))

#给urllib2安装opener
urllib2.install_opener(opener)

#使用带有cookie的urllib2访问网页
response = urllib2.urlopen(“http://www.baidu.com“)

python-mac_mysql链接出错

在初学python的时候链接MySql报出了SyntaxError: Non-ASCII character ‘\xe5’ in file的错误，最后查找各种原因，这种错误的原因是python的字符集和mysql的不一致导致的，python默认的字符集是ASCII码，而mysql我们一般设置的是utf8，所以需要在python脚本的第一行添加#-- coding: UTF-8 --，注意，一定要是在第一行

在MySQL-python的安装包路径下找到site.cfg文件，打开之后
# The path to mysql_config.
# Only use this if mysql_config is not on your PATH, or you have some weird
# setup that requires it.
# mysql_config = /usr/local/bin/mysql_config

将最后一行去掉注释，路径改为本机的路径：/usr/local/mysql/bin/mysql_config

然后再次执行：python setup.py install
此时可能会报错，如图

图中的意思是权限问题，此时使用sudo或者root权限再次执行上面的命令即可
sudo python setup.py install

MySQL-mysql5,7 for Mac 10.11安装

MySQL下载地址：http://dev.mysql.com/downloads/mysql/

目前mysql提供了两种安装方式
一：dmg/pkg，这两种安装包跟windows下的exe一样，都是傻瓜式安装，并没有什么技术含量，这里就不一一讲解
二：tar包安装，这也是本文要讲的主要内容
前提：
1.下载好mysql的tar包，并解压好
2.mac的当前拥有sudo权限，或者可以使用root权限（我是直接使用了root权限，而mac默认没有root用户，需要创建，点击详情）

步骤：
1.进入解压后的二进制文件所在的目录，将其移动到安装目录（*第一个路径是bin文件夹的上一级目录）
mv mysql-5.7.13-osx10.11-x86_64 /usr/local/mysql
2.更改mysql安装目录所属的用户和用户组
cd /usr/local
sudo chown -R root:wheel mysql
3.切换到mysql安装目录并执行初始化命令，并记录生成的临时root密码
cd /usr/local/mysql
bin/mysqld –initialize –user=mysql
(执行完成之后会在终端最后一行打印出：[Note] A temporary password is generated for root@localhost:XXXXXXX,最后的XXXXXXX即为临时密码，保存下来）
4.测试启动/重启/停止/查看运行状态
启动：
support-files/mysql.server start
重启：
support-files/mysql.server restart
停止：
support-files/mysql.server stop
检查mysql运行状态
support-files/mysql.server status
（下面有我写的shell脚本来管理mysql的启停）
5.修改root密码
进入bin目录，使用命令进入mysql操作环境
cd /usr/local/mysql/bin
./mysql -u root -p
输入第三步保存的密码
SET PASSWORD FOR ‘root’@’localhost’ = PASSWORD(‘newpass’);
之后就可以用新的密码来登录

SQL注入-入门

SQL注入（sql injection）
本例以J2EE为例

先做一下准备工作
开发工具：eclipse Mars，MySql，tomcat（eclipse Mars自带的tomcat也行）
调试工具：随便来个浏览器就行
相关知识：servlet，java连接数据库，数据库查询，sql语句，GET/POST请求原理
相关知识有任何问题建议百度谷歌走一遭

先讲一下我们登录的一般流程：
当我们使用浏览器进入某些网站输入账号密码登录的时候，
浏览器会通过GET或者POST请求（GET请求将账号密码放在在URL中，POST请求将账号密码放在请求体中，绝大多数以form表单形式post给服务器）
将我们的账号密码发送给服务器（某些网站会对用户信息进行加密上传，而有些不会），
服务器接收到我们的账号密码之后就会形成一个条sql语句，从数据出查询是否存在该账号与密码
如果存在就会提示登录成功进入下一个界面，如果不存在那么就错误会提示账号密码错误

那么我们说好的sql注入该怎么用？
先下载一下我写的服务器代码，在eclipse上运行（我是用的是eclipse mars，里面自带tomcat），服务器代码的git地址：
https://github.com/loginSin/SQLInjectionServer

我们以GET为例（因为我比较懒没有画界面），在地址栏输入账号密码点击登录之后，会形成一个URL：
http://localhost:8080/SQLInjection/login?account=123&password=123
之后服务器进入servlet会获取到account和password的值
String account = request.getParameter(“account”);
String password = request.getParameter(“password”);
拿到浏览器传来的账号密码，服务器形成一条这样的sql语句，开始从数据查询数据
select * from user where account = 123 and password =123
如果执行完sql之后得到相应的用户数据，那么就登录成功，否则失败，这个是最一般的流程

接下来我们开始注入
“–”的注释作用
将上面的url添加一个“–”，这么写
http://localhost:8080/SQLInjection/login?account=123--&password=123
我们会惊奇的发现也是登录成功，原因是服务器拿到数据之后拼接成的sql语句变成了
select from user where account = 123– and password =123
因为“–”在sql语句中意思是注释掉后面的内容，所以上面的sql的真正表达是
select from user where account = 123
意思是查找账号为123的用户是否存在，直接忽略掉了后半段的查找密码的过程

“or 1=1”的作用
如果url这么写
http://localhost:8080/SQLInjection/login?account=123 or 1=1&password=123
也会登录成功，sql语句变成了
select * from user where account = 123 or 1=1 and password =123
也会把后半段的验证密码给忽略掉

具有类似功能的还有
or 1=1–
or ‘a’=’a’

如果某些网站以上介绍的都不行，有可能是网站对sql注入进行了防范，可能需要进行一些特殊的处理
or 1=1
‘ or 1=1–
“ or 1=1–
‘ or ‘a’=’a’
“ or ‘a’=’a’
‘) or (‘a’=’a’
如果还不行，我建议还是换个，或者自己搭建一个服务器

本人这方面也是小白，SQL注入的内容也是查看网上相关内容自学，有什么问题可以相互讨论

服务端代码：https://github.com/loginSin/SQLInjectionServer

附录：
sql的注释有三种/*/和–和#
1）/需要和/配对使用，不好控制/是的位置，所以不常用
2）适用于Microsoft SQL Server，而MySql用–后面要加空格
当使用–后跟空格的时候，在url的转义为–%20
3）使用#来注释MySql
