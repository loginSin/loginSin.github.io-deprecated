MySQL-mysql5,7 for Mac 10.11安装

MySQL下载地址：http://dev.mysql.com/downloads/mysql/

目前mysql提供了两种安装方式
一：dmg/pkg，这两种安装包跟windows下的exe一样，都是傻瓜式安装，并没有什么技术含量，这里就不一一讲解
二：tar包安装，这也是本文要讲的主要内容
前提：
1.下载好mysql的tar包，并解压好
2.mac的当前拥有sudo权限，或者可以使用root权限（我是]直接使用了root权限，见：）
步骤：
1.进入解压后的二进制文件所在的目录，将其移动到安装目录（*第一个路径是包含bin文件夹的上一级目录）
mv mysql-5.7.13-osx10.11-x86_64 /usr/local/mysql
2.更改mysql安装目录所属的用户和用户组
cd /usr/local
sudo chown -R root:wheel mysql
3.切换到mysql安装目录并执行初始化命令，并记录生成的临时root密码
cd /usr/local/mysql
bin/mysqld –initialize –user=mysql
(执行完成之后会在终端最后一行打印出：[Note] A temporary password is generated for root@localhost:XXXXXXX,最后的XXXXXXX即为临时密码，保存下来）
4.测试启动/重启/停止/查看运行状态
启动：
support-files/mysql.server start
重启：
support-files/mysql.server restart
停止：
support-files/mysql.server stop
检查mysql运行状态
support-files/mysql.server status

bash脚本：

#!/bin/bash

function mStart(){
echo “mysql is starting”
sudo /usr/local/mysql/support-files/mysql.server start
}

function mStop(){
echo “mysql is stopping”
sudo /usr/local/mysql/support-files/mysql.server stop
}

function mRestart(){
echo “mysql is restarting”
sudo /usr/local/mysql/support-files/mysql.server restart
}

function mStatus(){
echo “looking mysql’s status”
sudo /usr/local/mysql/support-files/mysql.server status
}

if [ $1 == “start” ]
then
mStart
elif [ $1 == “stop” ]
then
mStop
elif [ $1 == “restart” ]
then
mRestart
elif [ $1 == “status” ]
then
mStatus
else
echo “please input start/stop/restart/status”
fi

参考：http://www.widlabs.com/article/mac-os-x-install-mysql-57-with-tar-gz

python-Mac配置python集成开发环境

1、下载mac版64为eclipse
下载链接：http://www.eclipse.org/downloads/
2、下载jdk并安装
下载链接：http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html
3、下载安装Pydev插件
下载链接：http://pydev.org/download.html
点击下面的Get zip releases下的SourceForge download
下载完成之后进行解压，然后将解压后的目录下的features和plugins两个目录下的内容复制到eclipse对应的文件夹中，选择合并复制
注：如果eclipse的是整个安装包，可以右击，选择“显示包内容”，并进入eclipse的文件夹下，再将对应内容复制到相应的文件夹中
4、eclipse配置python开发环境
打开eclipse->偏好设置->输入python，选择Python Interpreter->点击右侧Advanced Auto Config->选择合适的python版本->apply
5、新建python工程
new->other->输入python，选择PyDev Project->给项目起名->如果没有src文件夹为其创建src文件夹->右击src文件夹，New->file->>选择刚建好的python项目，并给文件命名为.py

注：windows下的配置与mac下的很类似，只不过第4不有点差异，在windows是在eclipse的windows->Perferences下进行配置，不再一一赘述

SQL注入-入门

SQL注入（sql injection）
本例以J2EE为例

先讲一下我们登录的一般流程：
当我们使用浏览器进入某些网站输入账号密码登录的时候，
浏览器会通过GET或者POST请求（GET请求将账号密码放在在URL中，POST请求将账号密码放在请求体中）
将我们的账号密码发送给服务器（某些网站会对用户信息进行加密上传，而有些不会），
服务器接收到我们的账号密码之后就会形成一个条sql语句，从数据出查询是否存在该账号与密码
如果存在就会提示登录成功进入下一个界面，如果不存在那么就错误会提示账号密码错误

那么我们说好的sql注入该怎么用？
先下载一下我写的服务器代码，在eclipse上运行（我是用的是eclipse mars，里面自带tomcat），服务器代码的git地址：
https://github.com/loginSin/SQLInjectionServer

我们以GET为例（因为我比较懒没有画界面），在地址栏输入账号密码点击登录之后，会形成一个URL：
http://localhost:8080/SQLInjection/login?account=123&password=123
之后服务器进入servlet会获取到account和password的值
String account = request.getParameter(“account”);
String password = request.getParameter(“password”);
拿到浏览器传来的账号密码，服务器形成一条这样的sql语句，开始从数据查询数据
select * from user where account = 123 and password =123
如果执行完sql之后得到相应的用户数据，那么就登录成功，否则失败，这个是最一般的流程

接下来我们开始注入
“–”的注释作用
将上面的url添加一个“–”，这么写
http://localhost:8080/SQLInjection/login?account=123--&password=123
我们会惊奇的发现也是登录成功，原因是服务器拿到数据之后拼接成的sql语句变成了
select from user where account = 123– and password =123
因为“–”在sql语句中意思是注释掉后面的内容，所以上面的sql的真正表达是
select from user where account = 123
意思是查找账号为123的用户是否存在，直接忽略掉了后半段的查找密码的过程

“or 1=1”的作用
如果url这么写
http://localhost:8080/SQLInjection/login?account=123 or 1=1&password=123
也会登录成功，sql语句变成了
select * from user where account = 123 or 1=1 and password =123
也会把后半段的验证密码给忽略掉

具有类似功能的还有
or 1=1–
or ‘a’=’a’

如果某些网站以上介绍的都不行，有可能是网站对sql注入进行了防范，可能需要进行一些特殊的处理
or 1=1
‘ or 1=1–
“ or 1=1–
‘ or ‘a’=’a’
“ or ‘a’=’a’
‘) or (‘a’=’a’
如果还不行，我建议还是换个，或者自己搭建一个服务器

开发工具：eclipse Mars，MySql，tomcat（eclipse Mars自带的tomcat也行）
调试工具：随便来个浏览器就行
相关知识：servlet，java连接数据库，数据库查询，sql语句，GET/POST请求原理
相关知识有任何问题建议百度谷歌走一遭

本人这方面也是小白，SQL注入的内容也是查看网上相关内容自学，有什么问题可以相互讨论

服务端代码：https://github.com/loginSin/SQLInjectionServer

附录：
sql的注释有三种/*/和–和#
1）/需要和/配对使用，不好控制/是的位置，所以不常用
2）适用于Microsoft SQL Server，而MySql用–后面要加空格
当使用–后跟空格的时候，在url的转义为–%20
3）使用#来注释MySql

python-文件和流-1

#打开文件：
open(name[.mode[.buffering]])
open函数使用一个文件名作为唯一的强制参数，然后放回一个文件对象。模式（mode）和缓存（buffering）参数都是可选的
eg:>> file = open(r”C:/file.txt”)

#文件模式
r 读模式
w 写模式
a 追加模式
b 二进制模式（可添加到其他模式中使用）

  • 读/写模式（可添加到其他模式中使用）

注：在UNIX中换行符是\n，而在windows中是\r\n，python会对此进行自动的转换

#读与写
文件的读写需要使用write和read方法（以字符串形式进行读写）
写eg：

f = open(“file.txt”,”w”)
f.write(“hello “)
f.write(“world !”)
f.close()

读eg：

f = open(“file.txt”,”r”)
f.read(4)
‘hell’
f.read()
‘0 world !’

详情请见：
Magnus Lie Hetland 《Python基础教程》（第2版.修订版）207~209

iOS开发-控件不能交互的原因

1>控件的userInteractionEnabled = NO
2>透明度（alpha值）小于等于0.01
3>控件被隐藏的时候，即hidden = YES
4>子视图的位置超出了父视图的有效范围
5>设置了需要交互的师徒，但是被其他开启了用户交互的视图盖住

iOS开发-响应者链条顺序

1>当APP启动以后创建UIApplication对象
2>然后启动“消息循环”监听所有事件
3>当用户触摸屏幕的时候，“消息循环”监听到这个触摸事件
4>“消息循环”首先把监听到的触摸事件传递给UIApplication对象
5>UIApplication对象再传递给UIWindow对象
6>UIWindow对象在传递给UIWindow的跟控制器（rootViewControlller）
7>控制器再传递给控制器所管理的view
8>控制器管理的view在其内部搜索本次触摸的店在哪个控件的范围内
9>找到对应的控件以后（调用该空间的touchesXXX方法），再依次向上返回，最终返回给“消息循环”
10>“消息循环”知道哪个空间被点击之后，再搜索该控件是否注册了对应的事件，如果注册了相应的事件，那么就会调用此点击事件

iOS开发-CocoaPods更新慢

CocoaPods有时候更新缓慢是因为使用pod install/pod update的时候会默认升级CocoaPods的spec仓库，所以导致更新缓慢
解决方案：
pod install –verbose –no-repo-update
pod update –verbose –no-repo-update
